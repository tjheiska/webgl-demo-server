About this demo

Description

This demo draws server generated 3D points with WebGL. The demo is real time since matrix rotation is also done by the server and sent to client using sockets. It may seem there is absolutely nothing useful in it, but this implementation is more like an engine, which can be used as a base for much more useful app. One possible usage could be 3D-map, where users could upload their own spatial data and use real-time server-side filtering when inspecting it.

The goal

  • To learn modern web development for mobile and desktop.
  • To have something to show to the recruiters.
  • To understand the performance capabilities of web apps with modern technologies and hardware.

What is inside?

  • WebGL, WebSockets (Socket.IO-interface), HTML5, Font Awesome, Bootsrap 4
  • Angular 2+ - with TypeScript source
  • VPS (Virtual Private Server) running Debian/Linux Stretch
  • Docker container 1 - Nginx http(s) server/proxy with 'Let's Encrypt' certificate (grade A+ from SSL Labs test)
  • Docker container 2 - Express-Node.js server written with TypeScript - with REST API and Real-Time Socket.IO control

Other stuff

WebGL-rendering is implemented as texture based instanced rendering. The author has used this technique before with OpenGL TBO (Texture Buffer Object) and wanted to try it with WebGL too.

XYZ-data is downloaded as PNG-compressed binary data, rendered to canvas to read the pixel data. This data is then interleaved (premultiplied alpha forced every fourth byte to 255).

This app is tested with modern version of following browsers: Chrome, Firefox, Edge and Internet Explorer.

Known issues

The code needs some refactoring. Specially server.ts and app.module.ts files need to be divided to multiple files. Also didn't get too familiar with Bootstrap 4 yet, so layout design is not bullet proof. For example the size of loader-modal is not optimal at the beginning and 'Info'-dropdown-functionality should be reconsidered.
